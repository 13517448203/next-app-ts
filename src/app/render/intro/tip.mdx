# 渲染
- 服务端渲染
  - 默认情况下，Next.js 使用服务器组件，可自动实现服务器渲染。
- 客户端渲染
  - 在 Next.js 中，客户端渲染是可选的，意味着必须明确决定 React 应在客户端上渲染哪些组件。
  - 使用客户端组件，需要在文件的顶部，在导入的上方，添加 React "[use client](https://react.dev/reference/react/use-client)"指令，用于声明服务器和客户端组件模块之间的边界。

## 服务器渲染
在 Next.js 中，有三种不同的服务器渲染策略：
- 静态渲染（默认）
  - 静态渲染是指在构建时生成页面的 HTML 文件，这意味着页面内容在部署时就已经准备好，并且可以直接提供给访问者。
  - 优点：更快的网站、减少服务器负载、SEO。
  - 缺点：无法使用动态数据、无法使用React的特性、不适合具有定期更新的个性化数据的仪表板。
- 动态渲染
  - 动态渲染是指在请求时生成页面的 HTML 文件，内容根据请求的参数或其他动态数据来生成。
  - 优点：实时数据。
  - 缺点：请求时会增加整体的响应时间、SEO(搜索引擎可能无法有效地抓取动态的内容)。
- 流媒体
  - 流式处理是一种数据传输技术，它允许您将路由分解为更小的“块”，并在它们准备就绪时逐步将它们从服务器流式传输到客户端，防止缓慢的数据请求阻塞整个页面。


## 客户端渲染
- 客户端渲染是指在浏览器中使用 JavaScript 动态生成页面的 HTML 内容。


## 服务器和客户端组合模式
- 服务器端和客户端组合模式指的是在同一个页面中同时使用服务器端渲染和客户端渲染来呈现内容。

## useClient、useServer
- [useClient](https://react.dev/reference/react/use-client)

- [useServer](https://react.dev/reference/react/use-server)

## 交错服务器和客户端组件
- 可以在服务器组件中导入客户端组件。
- 不支持在客户端组件中导入服务端组件，但支持将客户端组件作为props传入客户端组件。
